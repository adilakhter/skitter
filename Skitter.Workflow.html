    Link to this type destination() View Source 
    destination() :: {workflow_identifier(), Skitter.Component.port_name()}

    Address a token can be sent to.

    Specified as the combination of a valid workflow identifier that refers to a component instance and the name of a valid port of that component.

    Link to this type proto_instance() View Source 
    proto_instance() ::
  {Skitter.Component.t(), any(),
   [{Skitter.Component.port_name(), [destination()]}]}

    Proto-instance type.

    A proto-instance is a tuple which contains the component it represents, its init argument (which can be anything), and a list of destinations, arranged by out port.

    Link to this type t() View Source 
    t() :: %Skitter.Workflow{
  instances: %{required(workflow_identifier()) => proto_instance()},
  sources: %{required(workflow_identifier()) => [destination()]}
}

    Workflow data structure.

    A workflow is defined as a combination of named sources and proto-instances.
